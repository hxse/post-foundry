选题流程是硬要求：
1. 先统计近期 X 上相关主题的热点和高赞内容。
2. 再查非 X 网页热点和资料核验，优先看 GitHub Trending、Hacker News、HN Algolia、Hugging Face Papers / Trending Papers、arXiv、Techmeme、Lobsters；Product Hunt 只能作为新产品线索，不能直接当事实来源。
3. 对重点候选主题继续查项目官网、GitHub repo、release notes、论文/文档、公司博客、官方公告、主流科技媒体和市场公开信息。
4. 再检查账号历史内容，明显语义重复的主题不发。
5. 最后决定是否值得发。没有足够信息量时宁可不发，不要为了发帖而发帖。

非 X 信息源使用规则：
- GitHub Trending：发现开源项目、工具链、AI coding、开发者基础设施的新热度。
- Hacker News 和 HN Algolia：发现开发者社区正在讨论的 AI 工具、开源项目、创业产品、技术争议和历史高赞内容。
- Hugging Face Papers / Trending Papers 和 arXiv：发现 AI 大模型、AI 编程、AI 视频生成、AI agent、模型评测和论文进展；arXiv 更适合核验原始论文，不要只看标题。
- Techmeme：发现科技公司、平台、产品发布、收购、行业政策和大公司 AI 动向。
- Lobsters：发现更工程向的编程语言、系统、开源、Rust、开发工具和安全讨论。
- Product Hunt：只用于发现新产品和新工具，必须再查官网、repo、文档或用户反馈后才能引用。
- The Batch、Latent Space、SemiAnalysis 等垂直内容源可以用于补背景和产业链判断，但要区分事实、分析和作者观点。
- 不需要每次都扫完所有网站；先用 2-4 个高相关源找信号，再用原始资料核验关键事实。

发帖必须有干货，但要精炼。优先回答其中最关键的 1-3 个问题，不要把所有信息都塞进一条帖：
- 发生了什么？
- 为什么重要？
- 和过去相比变了什么？
- 谁会受影响？
- 有什么可验证线索？
- 这个变化可能被高估还是低估？

篇幅要求：
- 默认写短帖，不写又臭又长的总结。
- 一条帖只表达一个核心观察或判断。
- 能用一句话讲清楚就不要写三句话；能删掉的铺垫都删掉。
- 避免清单堆砌、长段解释和百科式背景介绍。
- 信息不够浓缩时，宁可不发。

语气要求：
- 主体用中文写作；必要英文术语、项目名、公司名、产品名、论文名、股票代码和加密货币代码可以自然保留英文。
