# Problem Context

PostFoundry 的发帖不能变成“随机让 AI 写一句话”。用户期望的运营方式是：

* 提取近期热点，来源包括公开 X 账号动态、近期高赞帖和新闻资料。
* 高赞帖只用于学习选题、表达节奏、信息密度和风格，不复制原文、不复述单条帖子的未经核验观点。
* 对热点重新查资料，优先使用可回看的资料和原始来源，发自然易懂的干货。
* 生成前先统计本账号近期发帖，明显语义重复的内容不能发。
* 内部落盘可以结构化，但真正进入 X official API 的 `post_text` 必须像真人自然发出的短句，不能有 Markdown、列表、标题、调试痕迹或模板味。

`.009` 先做离线 contract：定义 AI 草稿运行需要的输入包、输出草稿、发帖前 gate 和 ledger 写入。后续任务再接真实热点采集、在线 LLM 和自动发帖 executor。
