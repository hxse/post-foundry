# Problem Context

`.009` 已经定义 AI 发帖草稿 package，`.010` 能把资料整理成某个 topic 的 source context，`.011` 定义了真实 adapter 的离线边界。

缺口是：当一个账号有一批公开 X 热点、高赞帖、网页资料和人工笔记时，系统还不知道“今天先写哪个题”。如果直接把所有素材丢给 draft pipeline，后续很难审计为什么选这个题、为什么跳过另一个题，也容易和近期已发内容语义重复。

`.012` 的目标是在不调用在线服务的前提下，先落一个可回放的选题层。它输出 `.010` / `.009` 继续使用的 `CandidateTopic`，同时把候选题、分数、证据、近期重复抑制和最终选择写入 ledger。
