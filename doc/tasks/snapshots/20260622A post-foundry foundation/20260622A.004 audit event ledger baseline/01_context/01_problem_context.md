# Problem Context

项目目标不是简单让 AI 能发帖，而是让 AI 可以持续、可控、可复盘地运营自有 X 账号。运营动作包括统计热点、查找资料、生成候选内容、做发帖决策、执行发帖、读回表现和复盘策略。

用户确认的关键原则：

* 更偏自动化：普通无链接帖子在策略允许时可以由 AI 自动发。
* 带链接帖子因为 API 成本和风险更高，后续应通过 Telegram bot 通知人工审批。
* 自动化可以推进，但所有 AI 动作和决策必须可追踪、可审计。
* 每个账号的主题、数据源、历史、预算、决策和动作都必须按 `account_uuid` 隔离，不能混在一起。

`.003` 已经提供 SQLite runtime baseline，但只有 job 和 API audit；它还不能回答“AI 为什么这么做”“看了哪些证据”“谁批准了链接帖”“动作是否越过账号边界”。`.004` 负责补上这层审计地基。
